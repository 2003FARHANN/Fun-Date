# Interactive Date Proposal Website

A responsive, seven-screen proposal experience built with a Python/Flask backend and a polished HTML/CSS/JavaScript interface.

## Exact flow

1. **Proposal:** the `No` button escapes mouse, touch, focus, and click attempts.
2. **Surprise:** “WAIT, YOU ACTUALLY SAID YES??”
3. **Schedule:** date and time selection.
4. **Food:** choose one food vibe.
5. **Confirmation:** “Glad you didn't say no,” with the selected date and time.
6. **One small fee:** a clearly fictional fee generated by Python.
7. **Finale:** a saved date summary with confetti.

The fee screen is a joke. The project never asks for or processes real payment/card information.

## Run on Windows

Open the project folder in VS Code. In the terminal:

```powershell
py -m venv .venv
.venv\Scripts\activate
python -m pip install -r requirements.txt
python app.py
```

Open:

```text
http://127.0.0.1:5000
```

Stop the server with `Ctrl + C`.

## Personalize it

Open `config.json`. You can change:

- recipient and sender names;
- every heading, sentence, emoji, and button label;
- available time and food options;
- background, card, text, and button colors;
- the fee generation mode and amount.

### Random fee

```json
"mode": "random",
"random_min": 199,
"random_max": 999,
"random_step": 50
```

Python selects a cryptographically strong random value from that range whenever the fee screen opens.

### Fixed/manual fee

```json
"mode": "fixed",
"fixed_amount": 499
```

Restart the Flask server after changing `config.json`.

## Saved answers

Successful confirmations are stored in:

```text
instance/proposal.sqlite3
```

To view them:

```powershell
flask --app app responses
```

The fee returned by the server is signed. A visitor cannot silently change the amount before saving the response.

## Push to GitHub

Create an empty GitHub repository, then run from this project folder:

```bash
git init
git add .
git commit -m "Build interactive proposal website"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPOSITORY.git
git push -u origin main
```

Do not commit a real `.env` file or `instance/proposal.sqlite3`—both are already ignored.

## Deploy live on Render

1. Push the project to GitHub.
2. Sign in to [Render](https://render.com/).
3. Choose **New → Blueprint**.
4. Connect your GitHub repository.
5. Render detects `render.yaml` and deploys the Docker service.
6. Open the generated `onrender.com` link and share it.

The included `/api/health` endpoint is used for deployment health checks. Free hosting filesystems can reset during restarts; use a persistent disk or managed database if permanent response history is important.

## Docker run

```bash
docker build -t proposal-website .
docker run --rm -p 5000:5000 -e SECRET_KEY="replace-with-a-long-secret" proposal-website
```

Then open `http://127.0.0.1:5000`.
